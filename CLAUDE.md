# 仓库协作约束

## 语言与公开边界

- 公开文档、代码注释、测试说明和 Git 提交信息使用简体中文。
- 代码标识符、协议字段、配置键、第三方名称和许可证原文遵循既有写法。
- 不提交真实本机绝对路径、Obsidian 私人项目内容、密钥、令牌、凭据、日志或用户数据。
- `config/` 只保存脱敏示例；测试使用临时目录和虚构值。

## 修改原则

- 修改前先读取相关 Skill、脚本、模板和测试，确认多客户端影响。
- 工作流核心保持工具无关；客户端差异放在 `adapters/` 或安装器映射中。
- Skill 保持简洁、自包含和渐进披露，不在多个 Skill 中复制同一流程。
- 新增危险写操作必须支持预览、精确目标、默认不覆盖和明确失败。
- 不以降低安全检查、删除测试或放宽断言的方式制造通过结果。

## 验证

至少运行：

```bash
python3 -m unittest discover -s tests -v
python3 scripts/validate.py
```

修改 Skill 时还应使用目标客户端提供的 Skill 校验器；无法执行的验证必须如实说明。
