# 架构与事实源

## 三层事实

### 代码仓库

保存代码、测试、公开文档和团队可见的协作约束。这里的内容应当可以安全提交和公开分发。

### Obsidian 项目工作区

保存个人 TODO、实施计划、长期约束、产品说明、交接材料和需要长期保留的小型产物。工作区可以由 OneDrive 或其他同步盘同步，但不属于代码仓库。

### 机器本地秘密

API Key、Token、密码和真实用户数据保存在系统凭据库或机器本地私有目录，不进入 Git 或 Obsidian。

## 工作流结构

```text
TODO -> 计划 -> 真实代码调查 -> 单目标实现 -> 验证 -> 审查 -> Git 收口
  ^                                                          |
  +----------------------- 状态回写与完成摘要 ----------------+
```

`TODO.md` 是未完成任务入口，但不是实现依据的全部。计划说明为什么做、做到哪里、如何验收；真实代码和运行实例决定计划中的事实是否仍成立。

## 配置契约

配置只解决“仓库对应哪个 Obsidian 目录”，不存储项目内容或秘密。

```toml
version = 1
vault_root = "/path/to/obsidian-vault"

[projects]
"/path/to/personal/repository" = "Myproject/repository"
"/path/to/company/product/repository" = "Rsit/product/repository"
```

`apw install` 只安装用户级规则和 Skill，不创建本配置。项目第一次初始化时由用户选择 Vault 和 Vault 内的完整项目路径，初始化 Skill 再写入仓库精确映射。没有映射表示项目尚未初始化，不能自动回退到 `Myproject`。

旧配置中的 `projects_root` 只用于识别历史默认工作区，以便通过初始化 Skill 安全迁移；它不再决定新项目位置。后续调整目录时同时移动工作区并原子更新映射，不能只改配置或只移动目录。

## 管理器状态

`apw` 的安装状态不属于项目事实源，保存在用户级状态目录：

```text
macOS/Linux：~/.local/state/agent-project-workflow/install.json
Windows：    %LOCALAPPDATA%\agent-project-workflow\state\install.json
```

它只记录管理器版本、已选客户端、托管目标、文件哈希和私有运行时版本，不保存凭据、Obsidian 内容或项目计划。状态用于识别漂移、修复、客户端移除和卸载所有权。

## 平台差异

- 管理器目录：macOS/Linux 遵循 XDG（`~/.local/share`、`~/.config` 等）；Windows 统一位于 `%LOCALAPPDATA%\agent-project-workflow\` 下的 `config`、`data`、`state`、`cache`、`bin` 子目录。
- `current` 版本指针：macOS/Linux 用符号链接；Windows 用目录联接（junction），普通用户即可创建，无需管理员权限。
- PATH 接入：macOS/Linux 在 `.zprofile`/`.bashrc` 写入托管区块；Windows 写入用户环境变量（注册表 `HKCU\Environment`），按值幂等增删，不截断。
- 启动器：macOS/Linux 为 `~/.local/bin/apw`（shell 脚本）；Windows 为 `bin\apw.cmd`。

## 发布与更新

稳定版本使用 GitHub Release 中的单文件 `apw.pyz`。Bootstrap 安装固定版本私有运行时；后续只有显式 `apw update` 才下载发布清单和新 ZipApp。

正常更新先校验 SHA-256、暂存新版并运行诊断，再切换 `current`。已有用户内容发生漂移时不会静默覆盖，而是重新进入冲突预览流程。

管理器不会自动检查更新。降级必须通过 `apw update --version <版本> --allow-downgrade` 明确指定目标，并完成独立确认；非交互环境还必须提供 `--confirm-downgrade`。

常规更新不在后台替换私有运行时。发布清单要求的 Python 或 uv 与当前安装不一致时，更新会停止并提示重新运行 Bootstrap，避免一半更新程序、一半更新运行时的不可回滚状态。
