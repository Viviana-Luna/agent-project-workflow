# 计划状态与元信息

目录仍然是人类查看状态的主要方式：

- `planning/计划中/`：已立项但尚未执行。
- `planning/执行中/`：正在实施或验收。
- `planning/已完成/`：完成摘要。

新计划建议添加最少量 frontmatter，供 `workflow_doctor.py` 检查：

```yaml
---
id: feature-short-name
status: planned
kind: task
todo: ""
---
```

字段含义：

- `id`：项目内稳定且唯一的变更标识。
- `status`：`planned`、`active`、`paused` 或 `done`。
- `kind`：`task` 表示可执行单目标；`program` 表示长期主线或参考计划。
- `todo`：对应 TODO 中的稳定标识；没有时留空。

只有 `kind: task` 且 `status: active` 的计划必须被 TODO 当前执行区引用。长期主线可以使用 `kind: program`，避免为了保留背景而伪装成当前任务。
